# 数据源接入与合规检查表

本文件不是法律意见，而是开发前的最低检查要求。每个来源都应形成一条可审计记录。

## 接入前

- [ ] 记录首页、目标栏目、RSS/API、robots.txt、条款和版权页 URL。
- [ ] 记录检查日期、检查人、允许的访问方式与存储范围。
- [ ] 优先选择官方 RSS/API；若使用 HTML，确认列表页和正文页均为公开页面。
- [ ] 确认无需登录、验证码、订阅、地区绕过或其他访问规避。
- [ ] 设置专属域名限速、超时、重试上限、最大响应大小与 User-Agent。
- [ ] 明确是否允许保存全文、保存多久、是否仅限内部研究。
- [ ] 明确图片、视频、图表和附件的许可与处理方式。

## 开发与测试

- [ ] 测试使用本地 fixtures，不让 CI 高频访问生产网站。
- [ ] 解析器只提取必要字段；不复制无关页面资产。
- [ ] 记录 canonical URL、抓取时间、原始发布时间、内容 hash 和解析器版本。
- [ ] HTML 经过清洗；阻止脚本、内嵌对象和危险链接进入前端。
- [ ] 对 RSS/聚合 API 的文章回到原始发布者 URL，不能把聚合器当作版权授权方。
- [ ] 通讯社转载识别首发来源，避免重复存储同一全文。

## 运行期间

- [ ] 使用 ETag/Last-Modified 和缓存，减少无效请求。
- [ ] 403、401、429、验证码或条款提示出现时自动停用并等待人工复核。
- [ ] robots.txt 或条款变化触发告警，不自动假定旧许可继续有效。
- [ ] 每月复核核心源，每季度复核全部源；COP31期间提高复核频率。
- [ ] 按来源执行全文留存清理，不影响元数据、摘要、链接和审计日志。
- [ ] 用户请求删除或来源撤稿时，能够定位并处理全部版本和推送记录。

## 输出与推送

- [ ] 摘要显著链接原文并显示发布者、作者和日期。
- [ ] 不使用原文图片作为默认配图；需要时只提供链接或确认许可。
- [ ] 不输出长段原文；引语保持必要、短小、可追溯。
- [ ] 事实、媒体观点、倡议组织立场、模型推断和建议分开标注。
- [ ] 付费内容只显示可合法获得的标题、公开摘要和链接。

## 特别注意的来源

- Guardian RSS：官方说明面向个人、非商业用途；公开或商业部署前复核 Open Platform 条款。
- Reuters、AP、POLITICO：默认只做发现与链接核验，不镜像全文或图片。
- Google News RSS、GDELT、Media Cloud、Event Registry：它们帮助发现文章，但不替代原发布者的内容许可。
- 需要验证码或反机器人验证的站点：停止自动化，不使用绕过方案。
- UNEP RSS：当前公开 XML 使用非标准字段，且个别响应含损坏条目；只能导入可单独通过 XML 校验的完整 `<item>`，解析器版本须保留 `recovered` 标记。
- GDELT DOC API：2026-07-17 首次接入连续返回 HTTP 429；保持有限退避和失败记录，不改用规避限流的代理或轮换地址。
- OpenClimateData NDC CSV：只作为 UNFCCC NDC Registry 的传输镜像；逐行校验 `unfccc.int` 文件 URL，缺日期、非官方 URL 和重复版本不进入正式档案。
- UNFCCC Decisions：当前仅为 COP21—COP30 高价值决定索引，不能在页面或对外材料中表述为“完整决定库”。
