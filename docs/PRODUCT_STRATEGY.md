# 产品定位与渠道策略

更新日期：2026-07-17。本文件描述首版产品判断，不构成法律意见。

## 一句话定位

本系统不是“又一个每日新闻摘要”，而是面向中国气候谈判工作者的**可追溯谈判情报台**：把跨语言新闻、官方文件、关键数字、各方立场和历史版本组织成可复核的行动线索。

## 为什么在通用大模型之后仍有空间

通用模型擅长临时检索和生成流畅摘要，项目应避开这个正面竞争，聚焦模型自身不会持续维护的四类资产：

1. **机构记忆**：十年政策与谈判文件版本、数字口径、立场变化和事件关系。
2. **证据纪律**：事实、媒体观点、模型推断和建议分区；数字能回到原表、单位和时间范围。
3. **持续运行**：来源健康、抓取失败、版权范围、简报版本和投递结果均可审计。
4. **可控架构**：模型、Agent 和渠道可替换；不配置模型时，确定性采集与基础简报仍然运行。

这四项合起来形成“小而深”的产品楔子：**谈判准备、数字核验、立场演化与会期态势**，而非泛气候资讯。

## 首版用户任务

- 领导在 3 分钟内知道今天最需要关注的 3 件事、关键数字和建议动作。
- 业务人员能展开事件，核对原始来源、独立来源数、官方来源数和置信度。
- 研究人员能按议题查看过去十年的文件版本、立场变化和指标口径。
- 管理员能增删来源、替换模型/Agent、切换发布渠道，而不改动主流水线。

## 渠道建议

### 推荐组合：移动网页 + 邮件提醒

这是首期最稳妥的主渠道。邮件在手机和电脑上均可访问，领导层熟悉；正文只放一屏摘要和关键数字，详细内容通过无需安装应用的移动网页打开。系统不依赖某个平台的群发策略，内部邮件系统还可保留审计和通讯录权限。

### 组织内增强：企业微信或钉钉

如果用户已在同一组织内，可接企业微信应用消息或钉钉自定义机器人。它们适合“提醒 + 链接”，不应成为唯一的数据承载面。钉钉开放平台提供自定义机器人 Webhook 接入文档：<https://open.dingtalk.com/document/orgapp/custom-robot-access>。

### 仅作高优先级提醒：短信

短信覆盖最广、对年长用户最友好，但成本、长度、许可、用户同意和频次治理要求更高，不适合作为日报全文渠道。2026 年修订的《通信短信息服务管理规定》要求相关服务依法许可、实名核验、留存发送接收记录，并规范商业短信同意、拒收、时段和频次：<https://www.miit.gov.cn/gyhxxhb/jgsj/cyzcyfgs/bmgz/xxtxl/art/2026/art_f729621047ec4c30bcdd5cd4101f9568.html>。

### 不建议首期依赖

- 个人微信/公众号群发：平台限制会直接影响送达，且难以做完整投递审计。
- 新增独立 App：安装和学习成本高，不符合领导层的低摩擦要求。
- 仅发送 PDF：适合归档和打印，但手机端阅读与深层交互较差，可作为附属格式。

## 建议路线图

1. **MVP（当前）**：46 个来源注册表、SQLite、演示事件、可视化情报台、Markdown 简报、邮件/Webhook 适配器。
2. **下一步**：接通 8 个 P0 RSS/API，建立真实的 24 小时事件流与来源健康监控。
3. **历史库一期**：先导入 UNFCCC 文件元数据、NDC 版本和 COP 决定，不从“十年全网新闻”开始。
4. **会期模式**：2026-10-15 起提高核心源频率，跟踪议程与文件哈希，支持突发提醒。
5. **组织试点**：选择 5—15 名真实用户，评估“被打开、被用于准备材料、数字被纠错、建议被采纳”，而非只看推送数。

## 成功指标

- 每条高优先级事件的官方/原始来源引用覆盖率。
- 关键数字及单位、基期、时间范围的可回溯率。
- 从事件发生到中文可用简报的延迟。
- 用户每周实际用于谈判准备或材料撰写的事件数。
- 人工纠错率、重复推送率和低价值事件占比。
